## 🔍 Key Decisions Made

### Decision: 4-Layer Consistency System Architecture
- **Why**: Gemini research revealed AI has NO true identity lock - single reference image insufficient
- **Technical Implication**: Every `generateImage()` call now constructs multi-layered prompts
- **Alternatives Rejected**:
  - Single prompt reinforcement (too weak)
  - Image-only references (AI ignores visual data inconsistently)
  - Pre-generation fine-tuning (not feasible with Gemini API)
- **Files Affected**: `types.ts`, `App.tsx`
- **Learning**: AI models require redundant, multi-format reminders for visual consistency

### Decision: Return Type Change for generateImage()
- **Why**: Need to store original prompts for regeneration UI debugging
- **Technical Implication**: Changed from `Promise<string>` to `Promise<{ imageUrl: string; originalPrompt: string }>`
- **Alternatives Rejected**: Global prompt cache (memory overhead), separate logging system
- **Files Affected**: `App.tsx` (line ~519), `ComicFace` interface in `types.ts`

### Decision: Hard Negatives in Consistency Instructions
- **Why**: Prevents AI drift by explicitly stating what NOT to include
- **Technical Implication**: `generateHardNegatives()` creates contrast features (e.g., "no bangs" if character has slicked hair)
- **Alternatives Rejected**: Positive-only reinforcement (tested, insufficient)
- **Files Affected**: `types.ts` (line ~123-145)

### Decision: Cumulative Choice Tracking
- **Why**: Users may reroll decision pages multiple times - need to avoid ALL previous choices
- **Technical Implication**: `previousChoices` appended to `ComicFace` after each reroll
- **Alternatives Rejected**: Single reroll only (bad UX), reset after one reroll
- **Files Affected**: `App.tsx` (line ~336, ~1253), `types.ts` (ComicFace interface)

### Decision: Novel Mode Batch Size = 3
- **Why**: Balances UX (not too frequent pauses) with user control (see results before committing)
- **Technical Implication**: `NOVEL_MODE_BATCH_SIZE` constant in `types.ts`
- **Alternatives Rejected**: 5 pages (too long between pauses), 1 page (too disruptive)
- **Files Affected**: `types.ts` (line ~25), `App.tsx` (line ~747)

## 🐛 Issues & Solutions

### Issue: AI Ignores Single Reference Image
- **Error**: Characters change appearance between panels despite reference images
- **Cause**: Gemini image understanding is inconsistent; single modality insufficient
- **Solution**: Implemented 4-layer system with text reinforcement at multiple prompt positions
- **Learning Logged**: Yes - research/gemini/consistency-research.md

### Issue: Novel Mode Batch Size Varied
- **Error**: Sometimes 3 pages, sometimes 5, based on decision page placement
- **Cause**: `getComicConfig()` calculated batches without mode context
- **Solution**: Added `isNovelMode` parameter, used `NOVEL_MODE_BATCH_SIZE` constant
- **Learning Logged**: No - straightforward fix

### Issue: Rerolled Choices Repeated
- **Error**: AI sometimes regenerated similar/identical choices on decision page reroll
- **Cause**: No memory of rejected choices in generation context
- **Solution**: Added `previousChoices` tracking through generation pipeline
- **Learning Logged**: No - expected behavior fix
