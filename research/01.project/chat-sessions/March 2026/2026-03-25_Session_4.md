## 🔍 Key Decisions Made

### Decision: Hybrid Multi-Model Architecture (Claude + Gemini)
- **Why**: Leverage Claude's superior text analysis and narrative reasoning while keeping Gemini for image generation
- **Technical Implication**: All text generation functions now try Claude first, fallback to Gemini on failure
- **Alternatives Rejected**:
  - Full migration to Claude would lose Gemini's image generation
  - Keeping only Gemini would miss out on Claude's narrative strengths
  - Hard switch without fallback would reduce reliability
- **Files Affected**: `App.tsx`, `claudeHelpers.ts`, `types.ts`

### Decision: Claude as Primary, Gemini as Fallback
- **Why**: Claude Sonnet 4.5 provides better character analysis and story structure reasoning
- **Technical Implication**: Each text generation function has try/catch with Gemini fallback
- **Alternatives Rejected**: Equal priority between models would add complexity; making Gemini primary would lose quality gains
- **Files Affected**: `App.tsx` (generateCharacterProfile, generateOutline, generateBeat)

### Decision: Smart Alert System for API Keys
- **Why**: Guide users to configure optimal setup without blocking functionality
- **Technical Implication**: Red alert for missing Gemini (required), amber tip for missing Claude (optional), dismissible "Use Gemini Only" option
- **Alternatives Rejected**:
  - Blocking modal would prevent app usage
  - No alert would leave users unaware of Claude benefits
  - Always-visible alert would be annoying
- **Files Affected**: `App.tsx`, `SettingsDialog.tsx`

### Decision: Mobile-First Button Grid Layout
- **Why**: Horizontal scrolling on mobile was poor UX; grid layout is more thumb-friendly
- **Technical Implication**: Changed from `flex-row overflow-x-auto` to `grid grid-cols-2 md:grid-cols-4`
- **Alternatives Rejected**: Keeping horizontal scroll would fail mobile usability; single column would waste space on tablet
- **Files Affected**: `Setup.tsx`

### Decision: Type Safety Helpers for AI Responses
- **Why**: AI models sometimes return objects instead of strings (especially with JSON mode)
- **Technical Implication**: Created `ensureString()` in claudeHelpers and `safeString()` in types to handle both string and object responses
- **Alternatives Rejected**: Runtime errors without normalization would crash generation; manual checking everywhere would be verbose
- **Files Affected**: `claudeHelpers.ts`, `types.ts`

## 🐛 Issues & Solutions

### Issue: AI Models Returning Objects Instead of Strings
- **Error**: Runtime crash when AI returns `{text: "content"}` instead of `"content"`
- **Cause**: Claude API sometimes returns structured objects; JSON mode can wrap responses
- **Solution**: Created `ensureString()` and `safeString()` helpers that check type and extract `.text` property if present
- **Learning Logged**: Always normalize AI outputs with type guards

### Issue: Claude API Key Optional but User Doesn't Know
- **Error**: Users wouldn't know Claude integration exists without UI guidance
- **Cause**: No visual indicator of optional Claude key
- **Solution**: Added amber-themed tip banner that's dismissible with "Use Gemini Only" button
- **Learning Logged**: Progressive enhancement needs discoverable UI hints

### Issue: Mobile Button Layout Horizontal Scroll
- **Error**: Users couldn't see all buttons without scrolling horizontally on mobile
- **Cause**: `flex-row overflow-x-auto` forced horizontal layout
- **Solution**: Changed to `grid grid-cols-2 md:grid-cols-4` for responsive 2x2 on mobile, 4-column on tablet+
- **Learning Logged**: Grid layouts are more thumb-friendly than horizontal scrolling

### Issue: Alert Banner Showing After Admin Login
- **Error**: Alert banner re-appeared even after admin provided server-side key
- **Cause**: Didn't check `isAdminLoggedIn` state when determining alert visibility
- **Solution**: Added `!isAdminLoggedIn` to alert banner display condition
- **Learning Logged**: Server-side auth should suppress client-side key prompts
