### claudeHelpers.ts - detectImageMimeType
```typescript
export function detectImageMimeType(base64Data: string): 'image/jpeg' | 'image/png' | 'image/gif' | 'image/webp' {
    const cleanData = base64Data.includes(',') ? base64Data.split(',')[1] : base64Data;
    if (cleanData.startsWith('iVBORw0KGgo')) return 'image/png';
    if (cleanData.startsWith('/9j/')) return 'image/jpeg';
    if (cleanData.startsWith('R0lGOD')) return 'image/gif';
    if (cleanData.startsWith('UklGR')) return 'image/webp';
    return 'image/jpeg';
}
```

## Technical Notes

### Claude API Image Content
The Claude API is strict about image mime types matching actual content. The `detectImageMimeType()` function checks base64 magic bytes:
- PNG files always start with `iVBORw0KGgo` (decodes to `\x89PNG`)
- JPEG files always start with `/9j/` (decodes to `\xFF\xD8`)
- GIF files always start with `R0lGOD` (decodes to `GIF`)
- WebP files always start with `UklGR` (decodes to `RIFF`)

### Batch-of-3 Pattern
The comic generates pages in batches of 3 for both Novel Mode and Outline Mode. This reduces:
- Character drift (AI forgets features between pages)
- Narrative inconsistency
- Visual style drift

By explicitly telling the AI which position in the batch the current page is, it can better structure mini-arcs.

### Character Description Bug
The `Persona` interface has two description fields:
- `desc` - Legacy field, always empty (not connected to UI)
- `backstoryText` - Active field, populated from "Character Description" textarea in Setup.tsx

The bug was that `generateCharacterProfile()` only checked `persona.desc`, meaning user-provided character descriptions were silently ignored during AI profile analysis. This affected both Novel Mode and Outline Mode.

### Batch Generation Investigation
User reported images returning "Success: No (empty)" in Outline Mode.

**Findings:**
- Outline Mode DOES use batches, but pattern is: pages 1-2 (awaited) → pages 3-5 (background)
- Users CAN navigate while later pages generate (first batch completes before UI shows)
- Empty responses with "Candidates: 1" indicate content filtering, not API errors
- Enhanced debugging will reveal `finishReason` and `safetyRatings` to pinpoint cause

**Batch Flow (from App.tsx lines 1664-1665):**
```typescript

generateBatch(config.INITIAL_PAGES + 1, config.BATCH_SIZE);  // Background (no await)
```

**Config values (from types.ts):**
- `INITIAL_PAGES = Math.min(2, storyLength)` = 2 for 6+ page stories
- `BATCH_SIZE = 3` for Outline Mode (not Novel Mode)
