### claudeHelpers.ts - detectImageMimeType
```typescript
export function detectImageMimeType(base64Data: string): 'image/jpeg' | 'image/png' | 'image/gif' | 'image/webp' {
    const cleanData = base64Data.includes(',') ? base64Data.split(',')[1] : base64Data;
    if (cleanData.startsWith('iVBORw0KGgo')) return 'image/png';
    if (cleanData.startsWith('/9j/')) return 'image/jpeg';
    if (cleanData.startsWith('R0lGOD')) return 'image/gif';
    if (cleanData.startsWith('UklGR')) return 'image/webp';
    return 'image/jpeg';
}
```

## Technical Notes

### Claude API Image Content
The Claude API is strict about image mime types matching actual content. The `detectImageMimeType()` function checks base64 magic bytes:
- PNG files always start with `iVBORw0KGgo` (decodes to `\x89PNG`)
- JPEG files always start with `/9j/` (decodes to `\xFF\xD8`)
- GIF files always start with `R0lGOD` (decodes to `GIF`)
- WebP files always start with `UklGR` (decodes to `RIFF`)

### Batch-of-3 Pattern
The comic generates pages in batches of 3 for both Novel Mode and Outline Mode. This reduces:
- Character drift (AI forgets features between pages)
- Narrative inconsistency
- Visual style drift

By explicitly telling the AI which position in the batch the current page is, it can better structure mini-arcs.
