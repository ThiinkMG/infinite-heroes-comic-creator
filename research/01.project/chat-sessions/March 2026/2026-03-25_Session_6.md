## 🔍 Key Decisions Made

### Decision: Hidden File Input Pattern
- **Why**: Provides better UX control over file selection trigger (custom button styling)
- **Technical Implication**: Used `useRef` to programmatically trigger file input click
- **Alternatives Rejected**: Native file input (poor styling control, inconsistent browser appearance)
- **Files Affected**: `GalleryModal.tsx`

### Decision: Base64 Data URL Storage
- **Why**: Maintains consistency with existing comic page image storage format
- **Technical Implication**: FileReader.readAsDataURL converts uploaded files to base64 strings
- **Alternatives Rejected**: Blob URLs (would not persist in state/exports), File objects (not serializable)
- **Files Affected**: `GalleryModal.tsx`, `App.tsx`

### Decision: Replace Button Placement
- **Why**: Provide replace functionality in both grid and expanded views for maximum flexibility
- **Technical Implication**: Two separate button implementations sharing same handler logic
- **Alternatives Rejected**: Expanded view only (less discoverable), context menu (more complex UX)
- **Files Affected**: `GalleryModal.tsx`

### Decision: State Management via Parent Handler
- **Why**: Gallery Modal remains presentational, App.tsx controls state mutations
- **Technical Implication**: `onReplaceImage` callback passed from App updates `comicFaces` array
- **Alternatives Rejected**: Local state in GalleryModal (wouldn't persist), direct state mutation (breaks React patterns)
- **Files Affected**: `App.tsx`, `GalleryModal.tsx`

## 🧠 Technical Implementation Details

### FileReader API Pattern
```typescript
const handleFileChange = (e: React.ChangeEvent<HTMLInputElement>) => {
  const file = e.target.files?.[0]
  if (!file || !file.type.startsWith('image/')) return

  const reader = new FileReader()
  reader.onloadend = () => {
    if (reader.result && typeof reader.result === 'string') {
      onReplaceImage?.(replacingFaceId, reader.result)
      setReplacingFaceId(null)
    }
  }
  reader.readAsDataURL(file)
  e.target.value = '' // Reset input for re-selection
}
```

### State Update Pattern
```typescript
// In App.tsx
<GalleryModal
  onReplaceImage={(id, newImageDataUrl) => {
    setComicFaces(prev => prev.map(face =>
      face.id === id
        ? {...face, imageDataUrl: newImageDataUrl}
        : face
    ))
  }}
/>
```

### Hidden File Input Pattern
```typescript
const fileInputRef = useRef<HTMLInputElement>(null)

// Trigger programmatically
const handleReplaceClick = (id: number) => {
  setReplacingFaceId(id)
  fileInputRef.current?.click()
}

// Hidden input
<input
  type="file"
  accept="image/*"
  ref={fileInputRef}
  style={{ display: 'none' }}
  onChange={handleFileChange}
/>
```
