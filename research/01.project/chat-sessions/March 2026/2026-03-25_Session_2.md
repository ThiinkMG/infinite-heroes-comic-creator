## Key Decisions Made

### Decision 1: Comic Fundamentals Type Architecture
- **Why**: Needed structured, type-safe approach to comic fundamentals to enable consistent AI generation and user control
- **Technical Implication**: Created comprehensive type system in `types.ts` that serves as source of truth for all fundamentals-related features
- **Alternatives Rejected**:
  - String-based fundamentals (no type safety, harder to validate)
  - Separate fundamentals file (harder to import, less discoverable)
- **Files Affected**: `types.ts`, `App.tsx`, `OutlineStepDialog.tsx`

### Decision 2: Helper Functions for Prompt Generation
- **Why**: Centralize comic fundamentals logic to avoid duplication and ensure consistency across generation
- **Technical Implication**: `generateFundamentalsPrompt()` and description functions provide reusable utilities for AI integration
- **Alternatives Rejected**:
  - Inline prompt generation in App.tsx (would scatter logic)
  - Separate utilities file (unnecessary file proliferation)
- **Files Affected**: `types.ts`, `App.tsx`

### Decision 3: Reroll Override Architecture
- **Why**: Enable granular user control over reroll parameters while maintaining sensible defaults
- **Technical Implication**: RerollOptions interface with optional overrides allows flexible composition in generation pipeline
- **Alternatives Rejected**:
  - Required fields (would force users to specify everything)
  - Hardcoded reroll rules (no user customization)
- **Files Affected**: `types.ts`, `RerollModal.tsx`, `App.tsx`

### Decision 4: Mode Selection as Separate Screen
- **Why**: Create clearer user experience by separating mode choice from configuration
- **Technical Implication**: New ModeSelectionScreen acts as gatekeeper before reaching Setup component
- **Alternatives Rejected**:
  - Mode toggle in Setup (confused feature and configuration)
  - Modal dialog for mode selection (less prominent)
- **Files Affected**: `App.tsx`, `Setup.tsx`, `ModeSelectionScreen.tsx`

### Decision 5: Responsive Bottom Sheet for Mobile Reroll
- **Why**: Respect mobile UX conventions and improve touch interaction experience
- **Technical Implication**: Media query-based layout switching in RerollModal with CSS and component structure adjustments
- **Alternatives Rejected**:
  - Single responsive modal (uncomfortable on small screens)
  - Desktop-only reroll functionality (excludes mobile users)
- **Files Affected**: `RerollModal.tsx`

## Technical Implementation Details

### Comic Fundamentals Generation Flow

```
App.generateOutline()
├── Generate initial outline with fundamentals prompt
├── Parse fundamentals per page
└── Store in outline state

OutlineStepDialog
├── Display fundamentals for each page
├── Allow user editing of values
└── Update outline state

App.generateBeat()
├── Receive page with fundamentals
├── Apply reroll overrides if present
├── Include fundamentals in beat prompt
└── Generate narrative with guidance

App.generateImage()
├── Receive beat + fundamentals
├── Apply shot type to framing prompt
├── Apply transition to composition
├── Apply layout to panel structure
└── Generate image with consistency
```

### Reroll Pipeline with Overrides

```
RerollModal
├── User selects overrides (shot type, balloon, flashback)
├── Pass to App.rerollPanel()

App.rerollPanel()
├── Create RerollOptions with user selections
├── Call generateBeat(beat, overrides)
├── Call generateImage(beat, image, overrides)
└── Update panel with results
```

### Mode Selection Flow

```
App Initial Render
├── Check showModeSelection state
├── If true: Show ModeSelectionScreen
│   ├── User selects Novel or Outline
│   ├── Show confirmation
│   └── Set mode in state, hide selection
├── Else: Show Setup
```

## Issues & Solutions

### Issue 1: Type Safety with Fundamentals Enums
- **Challenge**: Ensuring fundamentals values are validated across generation pipeline
- **Solution**: Defined strict enums in types.ts, created type guards, used in switch statements for exhaustiveness checking
- **Resolution**: Prevents invalid values from reaching AI prompts

### Issue 2: Mobile Responsiveness for Reroll Modal
- **Challenge**: Reroll controls felt cramped on small screens
- **Solution**: Implemented bottom sheet pattern with media queries, responsive grid layouts
- **Resolution**: Comfortable touch interaction on all device sizes

### Issue 3: Backward Compatibility with Existing Outlines
- **Challenge**: Adding fundamentals to outline generation could break existing draft loading
- **Solution**: Made ComicFundamentals optional in outline parsing, provided sensible defaults
- **Resolution**: Existing drafts continue to work, new outlines have fundamentals

### Issue 4: Character Consistency with Fundamentals Context
- **Challenge**: More context in generation could confuse AI about character focus
- **Solution**: Enhanced character profile prompts with explicit focus character designation
- **Resolution**: Fundamentals enhance rather than distract from character consistency
