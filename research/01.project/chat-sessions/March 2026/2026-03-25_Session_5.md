## 🔍 Key Decisions Made

### Decision: Consistent Copy/Download Pattern
- **Why**: Story Outline and Original Prompt are both debug/utility features that benefit from both clipboard and file export
- **Technical Implication**: Users can quickly paste prompts into external tools OR save for later analysis
- **Alternatives Rejected**:
  - Copy-only (users requested download capability)
  - Download-only (copy is faster for quick sharing)
- **Files Affected**: `RerollModal.tsx`

### Decision: Page-Specific Filename for Original Prompt
- **Why**: When debugging multiple pages, generic "original-prompt.txt" would cause filename conflicts
- **Technical Implication**: Downloads now generate `page-5-original-prompt.txt`, `page-12-original-prompt.txt`, etc.
- **Alternatives Rejected**:
  - Generic filename (would overwrite previous downloads)
  - Timestamp-based (harder to identify which page)
- **Files Affected**: `RerollModal.tsx` (Line 407)

### Decision: Flex-Wrap for Button Container
- **Why**: On smaller RerollModal widths, header + two buttons could cause overflow
- **Technical Implication**: Buttons wrap to next line if needed, preventing layout breaks
- **Alternatives Rejected**: Fixed width button container (could clip text)
- **Files Affected**: `RerollModal.tsx` (Line 379)

### Implementation Details

```tsx
// Copy button pattern (reused in both sections)
onClick={() => {
    navigator.clipboard.writeText(content);
    const btn = document.activeElement as HTMLButtonElement;
    const originalText = btn.textContent;
    btn.textContent = '✓ Copied!';
    setTimeout(() => { btn.textContent = originalText; }, 1500);
}}
```

```tsx
// Download button pattern (new in this session)
onClick={() => {
    const blob = new Blob([originalPrompt], { type: 'text/plain' });
    const url = URL.createObjectURL(blob);
    const a = document.createElement('a');
    a.href = url;
    a.download = `page-${pageIndex}-original-prompt.txt`;
    a.click();
    URL.revokeObjectURL(url);
}}
```
