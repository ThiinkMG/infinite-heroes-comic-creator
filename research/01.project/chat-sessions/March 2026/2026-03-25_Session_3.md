## Key Decisions Made

### Decision: Comprehensive Comic Fundamentals Type System
- **Why**: Enable AI to reason about comic theory explicitly and provide structured page planning with visual indicators
- **Technical Implication**: Added 9 new TypeScript interfaces (TransitionType, ShotType, PanelLayout, EmotionalBeat, etc.) and 7 helper functions for prompt injection
- **Alternatives Rejected**: Simpler approach without enum-based types would reduce flexibility; flat string approach would lose type safety
- **Files Affected**: `types.ts`, `App.tsx`

### Decision: Visual Page Card UI in OutlineStepDialog
- **Why**: Make comic fundamentals visible and intuitive at a glance during outline review; support grid + text dual-view modes
- **Technical Implication**: Redesigned entire OutlineStepDialog layout with CSS Grid, icon mapping, and toggle state management
- **Alternatives Rejected**: Text-only summary would be less intuitive; single-view approach would reduce flexibility
- **Files Affected**: `OutlineStepDialog.tsx`

### Decision: Helper Functions for Prompt Injection
- **Why**: Reduce AI generation code duplication and provide consistent formatting for comic fundamentals guidance
- **Technical Implication**: Created getLayoutInstructions(), getShotInstructions(), getTransitionContext(), getFlashbackInstructions() for reusable prompt patterns
- **Alternatives Rejected**: Inline string concatenation would be harder to maintain; centralized helper approach is cleaner
- **Files Affected**: `types.ts`

### Decision: PageCharacterPlan as Central Narrative Structure
- **Why**: Unite all comic fundamentals (layout, shot, transition, emotional beat, pacing) into single queryable unit aligned to story beats
- **Technical Implication**: New parseEnhancedOutline() function converts AI outline output into PageCharacterPlan[] array for downstream generation
- **Alternatives Rejected**: Separate storage of each fundamental would create synchronization issues
- **Files Affected**: `App.tsx`, `types.ts`

## Issues & Solutions

### Issue: Parsing AI-Generated Outline into Structured Data
- **Error**: AI output format inconsistent, required flexible parsing strategy
- **Cause**: Large language models generate varied text formats; needed robust parsing
- **Solution**: Implemented parseEnhancedOutline() with regex-based extraction and fallback defaults for missing values
- **Learning Logged**: Flexible enum-based defaults improve resilience

### Issue: Balancing Comic Theory Complexity with Practical UI
- **Error**: Too much visual information on page cards caused clutter
- **Cause**: Tried to show all 9 comic fundamentals simultaneously
- **Solution**: Designed card with 4 key indicators (layout icon, shot type, beat color, pacing indicator) plus expandable details
- **Learning Logged**: Progressive disclosure improves UX for complex domains

### Issue: Maintaining Type Safety Across Comic Fundamentals
- **Error**: Enum values inconsistent between prompt injection and type definitions
- **Cause**: Manual string copying between multiple files
- **Solution**: Centralized enum definitions in types.ts with helper functions that reference enums directly
- **Learning Logged**: Single source of truth prevents sync errors
