## 🔍 Key Decisions Made

### Decision: Emblem Placement as Enum + Custom Option
- **Why**: Covers 99% of superhero emblem positions while allowing flexibility
- **Technical Implication**: UI dropdown with 8 presets + "Other (specify)" option
- **Alternatives Rejected**: Free-form text only (too vague for AI), fixed positions only (too rigid)
- **Files Affected**: `types.ts`, `Setup.tsx`, `App.tsx`
- **Implementation**:
  ```typescript
  export type EmblemPlacement =
    | 'chest-center' | 'chest-left' | 'chest-right'
    | 'back' | 'face' | 'hair'
    | 'shoulder' | 'weapon' | 'other';

  // Persona interface
  emblemImage?: string;
  emblemPlacement?: EmblemPlacement;
  emblemPlacementCustom?: string; // for 'other'
  ```

### Decision: Weapon Description as Optional Text Field
- **Why**: Allows users to specify unique weapon properties beyond visual reference
- **Technical Implication**: Combined visual + text description for maximum AI understanding
- **Alternatives Rejected**: Image-only (misses context like "glowing" or "cursed"), text-only (misses visual details)
- **Files Affected**: `types.ts`, `Setup.tsx`, `App.tsx`
- **Example Usage**: Image of sword + "Glowing ethereal blade that phases through solid matter"

### Decision: Full-Screen Modal for Backstory Editing
- **Why**: Character backstories often exceed 500+ words, cramped textarea frustrates users
- **Technical Implication**: Z-50 overlay modal with large textarea, character count display
- **Alternatives Rejected**: Inline expansion (breaks layout), side panel (loses context), tooltip (too small)
- **Files Affected**: `Setup.tsx`
- **Implementation Details**:
  - Responsive: `w-full max-w-4xl mx-4 sm:mx-auto`
  - Large textarea: `min-h-[400px] sm:min-h-[500px]`
  - Character count: Live display below textarea
  - Click outside or X button to close

### Decision: Negative Prompt in RerollModal
- **Why**: Users need granular control to exclude specific unwanted elements
- **Technical Implication**: Red-highlighted textarea sends exclusions to AI with strong "DO NOT INCLUDE" instruction
- **Alternatives Rejected**: Checkboxes (can't cover all cases), auto-detection (unreliable)
- **Files Affected**: `types.ts`, `RerollModal.tsx`, `App.tsx`
- **Prompt Format**: "DO NOT INCLUDE THESE ELEMENTS: {negativePrompt}"

### Decision: Enhanced AI Character Analysis (Mask + Hair + Weapon)
- **Why**: Previous analysis missed critical visual elements causing consistency issues
- **Technical Implication**: Extended `CharacterProfile` with `maskDescription`, `hairDetails`, `weaponDescription`
- **Alternatives Rejected**: Manual entry only (tedious), ignore these elements (poor consistency)
- **Files Affected**: `types.ts`, `App.tsx`
- **AI Prompt Addition**:
  ```
  - MASK: Describe any mask, helmet, face covering, or headgear in detail...
  - HAIR: Describe hair length, type, color, style, distinctive features...
  - WEAPON: Describe any signature weapon, tool, or held object...
  ```

### Decision: Hard Negatives for Mask/Helmet Detection
- **Why**: AI often hallucinates helmets/masks on characters who don't have them
- **Technical Implication**: If no mask detected in reference, add "NO mask, NO helmet, NO face covering" to hard negatives
- **Alternatives Rejected**: Ignore (consistency breaks), always add negative (prevents legitimate masks)
- **Files Affected**: `App.tsx`
- **Logic**:
  ```typescript
  if (!maskDescription || maskDescription.toLowerCase().includes('no mask')) {
    hardNegatives.push('mask', 'helmet', 'face covering', 'visor');
  }
  ```
